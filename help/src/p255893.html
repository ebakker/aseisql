Create a new job, schedule or scheduled job.

sp_sjobcreate name, option

Parameter information :
»name : The name argument is the name of a scheduled job. To specify the name of a job or a schedule, the name argument is prefixed with jname or sname (as in "jname=...")
»option : A list separated by commas of the field names and values you use to create a job, a schedule, or a scheduled job. Possible values: dates : a list of dates in a month, from 1 to 31, separated by colons. The number 32 represents the last day of any month., days : a list of days separated by colons. Full names or abbreviations in the locale of the server may be used., default_timeout, enable : Enable or disable, values to use are true, false, 0 and 1., enddate : the date on which a scheduled job ends., endtime : the time of day on which a scheduled job ends., jcmd : the SQL text. This is used for simple jobs when it is easy to provide the text directly., jdesc : comments describing the job., jdescription : comments describing the job (same as jdesc), jname : the name of the job, which must be unique., jowner, jproperties, owner, properties : the properties of the job, the schedule, or scheduled job: jproperties, sproperties, or sproperties, respectively., repeats : the interval at which the schedule repeats. 0 or NULL means it does not repeat. The value is a number followed by: day or d, days or dd, hour or h, hours or hh, minute or m, minutes or mm., sdesc : comments describing the schedule., sdescription : comments describing the schedule (same as sdesc)., server : the name of the server where the job runs. The default is the local server., sjproperties, sname : the name of the schedule, which must be unique., sowner, sproperties, startdate : The date on which a scheduled job starts., starttime : the time of day when a scheduled job begins., timeout, tmpl_id
